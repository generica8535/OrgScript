# docs(examples): curate the example catalog

## Why

OrgScript adoption depends on recognizable examples more than on new language features.

## Scope

- organize examples into `simple`, `realistic`, and `advanced`
- curate at least three strong example themes:
  - sales / lead flow
  - approval / finance
  - service / escalation
- make it easier to understand which examples are introductory and which are edge-case oriented

## Definition Of Done

- example browsing is easier than today
- example categories are documented
- the catalog shows both learning-oriented and realistic usage patterns

## Labels

- `v0.5.0`
- `documentation`
- `examples`
- `adoption`

## Dependencies

- none

## Notes

Prefer a smaller, stronger example set over a larger, unstructured one.
