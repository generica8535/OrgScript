# GitHub Labels

These labels provide a small, stable vocabulary for OrgScript issue triage and contributor onboarding.

## Core contributor labels

| Label | Purpose |
| --- | --- |
| `good first issue` | approachable, well-scoped tasks for first-time contributors |
| `help wanted` | tasks where outside help is actively welcome |
| `documentation` | README, guides, examples, or reference docs |
| `editor` | editor support such as syntax highlighting or extension scaffolds |
| `dx` | developer experience and contributor ergonomics |
| `diagnostics` | human-readable and machine-readable diagnostics |
| `cli` | command-line behavior, flags, and output |
| `quality` | combined checks, CI paths, or release-hardening work |
| `governance` | language evolution, compatibility, and change-management guidance |

## Suggested starter label set

If you want a very small public label surface, start with:

- `good first issue`
- `help wanted`
- `documentation`
- `editor`
- `dx`
- `cli`

## Suggested usage

- use `good first issue` only for tasks that a new contributor can finish without understanding the full language core
- use `help wanted` for issues that are valuable but not blocked on internal architecture work
- combine one or two focus labels to describe the work area
- avoid labeling vague or underspecified work as `good first issue`
