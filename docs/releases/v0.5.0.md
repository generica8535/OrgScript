# OrgScript v0.5.0

OrgScript v0.5.0 is the first adoption and polish release.

This release focuses on making OrgScript as easy as possible to try, read, and understand. Instead of expanding the language, we improved the developer experience at the "front door"—our README, examples, diagnostics, and editor highlighting.

## Highlights

- **Visual Showcase**: A revamped `README.md` showing the "Three-Value Flow" from `.orgs` source to visual artifacts.
- **60-Second Quickstart**: Faster path from installation to first successful command.
- **Improved Highlighting**: Refined TextMate grammar for VS Code, including inline keywords and dotted properties.
- **Curated Examples**: Reorganized example catalog categorized by learning depth (Simple, Realistic, Advanced).
- **Polished Diagnostics**: More readable CLI reports with consistent padding, bracketed codes, and "passed" status messaging.
- **Language Evolution Governance**: Documented boundaries between core language growth and tooling-level extensions.

## CLI and UX

- Updated `check`, `lint`, and `format --check` to use `passed` instead of `ok` in human-readable output.
- Refined diagnostic line formatting: `  SEVERITY [code] file:line - message`.
- Added consistent padding to severity levels for better vertical alignment in logs.
- Synchronized the diagnostics text contract in `spec/diagnostics.md` with the new CLI behavior.

## Documentation and Onboarding

- Added a visual showcase in the root `README.md`.
- Added categorized example descriptions in `examples/README.md`.
- Added `docs/language-evolution.md` for clear project governance.
- Improved VS Code extension onboarding and local testing documentation.

## Editor Support

- Refined `editors/vscode/syntaxes/orgscript.tmLanguage.json` to handle keywords like `then` and `else if` without requiring line-start positioning.
- Improved coloring for dotted references and operators.
- Cleaned up scope naming for better compatibility with modern VS Code themes.

## Verification

Verified with:

- `npm test`
- `npm run check:all`
- `node ./bin/orgscript.js check ./examples/craft-business-lead-to-order.orgs`
- `node ./bin/orgscript.js export mermaid ./examples/order-approval.orgs`
- `node ./bin/orgscript.js export markdown ./examples/order-approval.orgs`

## Project Status

v0.5.0 ensures that OrgScript is not just a stable specification, but a toolset that invites exploration. By focusing on polish and adoption readiness, we have made the project's value proposition visible at first glance.
