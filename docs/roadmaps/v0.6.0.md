# OrgScript v0.6.0 Roadmap

Status: Planned

## Theme

v0.6.0 focuses on integrations, generated artifacts, and showcase value.

The goal is to make OrgScript easier to demonstrate, easier to consume downstream, and more obviously useful in real workflows.

This is not a language-expansion release.
It is an artifact and adoption release.

## Goals

- strengthen OrgScript as a source-to-artifact workflow
- improve generated outputs
- make demos and examples more compelling
- increase downstream usability for documentation and integrations
- keep the core language stable while improving surrounding value

## Planned work packages

### WP1 — Mermaid Export Polish
Improve Mermaid output quality and consistency for both `process` and `stateflow`.

Scope:
- review generated Mermaid readability
- improve node/edge naming consistency
- improve layout-friendliness where possible
- tighten deterministic output behavior
- extend golden coverage if needed

Expected result:
Mermaid export becomes more presentation-ready and easier to embed in docs.

---

### WP2 — Markdown Summary v2
Improve the generated Markdown summaries so they work better as lightweight documentation artifacts.

Scope:
- improve summaries for multi-block files
- make section ordering more deliberate
- improve consistency across block types
- keep the output concise and deterministic
- avoid turning summaries into free prose

Expected result:
Markdown export becomes more useful as reviewable documentation.

---

### WP3 — HTML Documentation Export
Add a first HTML documentation export path.

Scope:
- `orgscript export html <file>`
- generate a simple, static HTML artifact
- include at least:
  - title
  - block sections
  - readable summaries
  - optionally embedded Mermaid source or rendered-friendly sections
- keep styling minimal and stable

Expected result:
OrgScript can generate a shareable documentation artifact beyond CLI and raw Markdown.

---

### WP4 — Demo and Showcase Upgrade
Turn demos into a stronger adoption asset.

Scope:
- improve demo structure under `docs/demos/`
- make source → Mermaid → summary → optional HTML flow explicit
- add one or two especially strong showcase cases
- ensure demo generation scripts stay aligned with actual exporter behavior
- improve README references to demos

Expected result:
The repository shows OrgScript’s value in a more immediate and compelling way.

---

### WP5 — Integration Contract Polish
Strengthen downstream usability for tooling and integrations.

Scope:
- review canonical model documentation
- review exporter boundaries vs core language
- tighten documentation for output stability where appropriate
- clarify which outputs are intended for human consumption vs machine integration

Expected result:
OrgScript becomes easier to integrate into external tooling and documentation pipelines.

## Explicit non-goals

v0.6.0 does not prioritize:
- new language keywords
- new block types
- major parser redesign
- multi-language syntax
- IntelliSense or full IDE language services
- SaaS/cloud features

## Success criteria

v0.6.0 is successful if:
- OrgScript produces stronger downstream artifacts
- demos become more convincing
- generated outputs are easier to use in real documentation workflows
- the language core remains stable while utility increases
